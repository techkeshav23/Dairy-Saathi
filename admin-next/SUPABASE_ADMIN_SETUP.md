# MY ORDER PRO admin — connect to Supabase (Phase 5)

The admin UI works on mock data out of the box. To show REAL data (same DB as the
mobile app):

1. Make sure you've created the Supabase project + run `../supabase/schema.sql`
   (see ../SUPABASE_SETUP.md).
2. `cp .env.local.example .env.local` and paste your Project URL + anon key.
3. `npm install` (pulls @supabase/supabase-js) then `npm run dev`.
4. In each page under `src/app/(app)/`, swap the static imports from `@/lib/data`
   to the async fetchers in `@/lib/supabase-data` (e.g. `await getOrders()` in the
   Server Component). `useSupabase` auto-falls back to mock when env is unset.
