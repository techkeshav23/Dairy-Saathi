# MY ORDER PRO — Supabase backend setup (Phase 4)

The app now supports a real Supabase backend. Until you plug in credentials it keeps
running on MockRepository, so nothing breaks.

## Steps (~10 min, free tier)

1. Go to https://supabase.com -> New Project. Pick a name + DB password + region (Mumbai
   for India). Wait ~2 min for provisioning.
2. In the project: **SQL Editor** -> paste the contents of `supabase/schema.sql` -> Run.
   (Optional: also run `supabase/seed.sql` to add sample rows.)
3. **Project Settings -> API** -> copy:
   - Project URL  (https://xxxx.supabase.co)
   - anon public key
4. Open `lib/data/supabase_config.dart` and paste both into `url` and `anonKey`.
5. `flutter pub get` then run the app. `SupabaseConfig.useSupabase` flips to true
   automatically and the app now reads/writes real data.

## How it's wired
- `lib/data/repository.dart` — the interface (unchanged).
- `lib/data/supabase_repository.dart` — real impl using supabase_flutter.
- `lib/data/repository_provider.dart` — picks Supabase vs Mock based on config.
- Anywhere the UI needs data, use `RepositoryProvider.instance` instead of
  `MockRepository()` directly. (Search the codebase for `MockRepository()` and swap
  to `RepositoryProvider.instance` — or wire it in your provider classes.)

## Auth (phone OTP)
- Enable Phone auth in Supabase -> Authentication -> Providers -> Phone (needs an SMS
  provider like Twilio/MSG91 for production; for testing use the Supabase test OTP).
